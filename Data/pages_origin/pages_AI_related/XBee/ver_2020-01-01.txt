'''Digi XBee''' is the brand name of a family of [[Computer form factor|form factor]] compatible radio modules from [[Digi International]]. The first XBee radios were introduced under the '''MaxStream''' brand in 2005<ref>{{cite web|url=http://www.freescale.com/files/abstract/press_release/MAXSTREAM_ZIGBEE_PR.html |title=Archived copy |accessdate=May 11, 2011 |url-status=dead |archiveurl=https://web.archive.org/web/20120316101443/http://www.freescale.com/files/abstract/press_release/MAXSTREAM_ZIGBEE_PR.html |archivedate=March 16, 2012 }}</ref> and were based on the [[IEEE 802.15.4]]-2003 standard designed for point-to-point and star communications at over-the-air baud rates of 250&nbsp;kbit/s.<ref>http://search.digikey.com/scripts/DkSearch/dksus.dll?Detail&name=XB24-AWI-001-ND</ref>

Two models were initially introduced&nbsp;— a lower cost 1&nbsp;mW '''XBee''' and the higher power 100&nbsp;mW '''XBee-PRO'''.<ref>https://www.youtube.com/watch?v=G_ScqGDY2eo</ref> Since the initial introduction, a number of new XBee radios have been introduced and an [https://www.digi.com/lp/xbee ecosystem] of wireless modules, gateways, adapters and software has evolved. 
 
The XBee radios can all be used with the minimum number of connections&nbsp;— power (3.3&nbsp;V), ground, data in and data out ([[Universal asynchronous receiver/transmitter|UART]]), with other recommended lines being Reset and Sleep.<ref>{{cite web|url=http://www.digi.com/support/kbase/kbaseresultdetl.jsp?id%3D2176 |title=Archived copy |accessdate=May 11, 2011 |url-status=dead |archiveurl=https://web.archive.org/web/20110722014704/http://www.digi.com/support/kbase/kbaseresultdetl.jsp?id=2176 |archivedate=July 22, 2011 }}</ref> Additionally, most XBee families have some other [[flow control (data)|flow control]], [[input/output]] (I/O), [[analog-to-digital converter]] (A/D) and indicator lines built in. A version called the programmable XBee has an additional on-board processor for user's code. The programmable XBee and a [[Surface-mount technology|surface-mount]] version of the XBee radios were both introduced in 2010.<ref>http://news.thomasnet.com/fullstory/Programmable-ZigBee-Module-eases-application-development-573823</ref>

==Form-factors, antennas, and data modes==
[[File:Pair of XBee Series 2s with Whip Antennas.jpg|thumb|A pair of XBee radios (through-hole with the wire whip antenna type).]]
XBee modules are available in three form-factors&nbsp;— [[Through-hole technology|through-hole]], surface mount (SMT), and micro-mount (MMT). All XBees, with the exception of the XBee 868LP, are available in the popular 20-pin through-hole form-factor. Certain XBee modules are also available in a 37-pad surface-mount design, which is popular for higher volume applications due to the reduced manufacturing costs of SMT.

XBee modules typically come with several antenna options, including [[Hirose U.FL|U.FL]], PCB embedded, wire, and [[SMA connector#Reverse polarity SMA|RPSMA]].

The XBees can operate either in a transparent data mode or in a packet-based [[application programming interface]] (API) mode.<ref name="XBeeRefGuide">http://www.tunnelsup.com/tup/2012/11/30/xbee-s2-quick-reference-guide-cheat-sheet/</ref> In the transparent mode, data coming into the Data IN (DIN) pin is directly transmitted over-the-air to the intended receiving radios without any modification. Incoming packets can either be directly addressed to one target (point-to-point) or broadcast to multiple targets (star). This mode is primarily used in instances where an existing protocol cannot tolerate changes to the data format. [[Hayes command set#Description|AT commands]] are used to control the radio's settings. In API mode the data are wrapped in a packet structure that allows for addressing, parameter setting and packet delivery feedback,<ref>{{cite web|url=http://www.digi.com/learningcenter/video/player.jsp?videoid%3Dzigbee-at-vs-api-commands |title=Archived copy |accessdate=May 11, 2011 |url-status=dead |archiveurl=https://web.archive.org/web/20110722014817/http://www.digi.com/learningcenter/video/player.jsp?videoid=zigbee-at-vs-api-commands |archivedate=July 22, 2011 }}</ref> including remote sensing and control of digital I/O and analog input pins.<ref name="Faludi2010">{{cite book|author=Robert Faludi|title=Building Wireless Sensor Networks: With ZigBee, XBee, Arduino, and Processing|url=https://books.google.com/books?id=xMC69vQJLZIC|accessdate=7 June 2012|date=24 December 2010|publisher=O'Reilly Media, Inc.|isbn=978-0-596-80773-3}}</ref>

== Product line ==
As of March 2016, the XBee radio family consists of<ref>http://www.digi.com/xbee/</ref>
* XBee 802.15.4&nbsp;— The initial [[Network topology#Point-to-point|point-to-point topology]] or [[Network topology#Star|star topology]] module running the [[IEEE 802.15.4]] protocol
* XBee-PRO 802.15.4&nbsp;— A higher power, longer range version of the XBee 802.15.4
* XBee DigiMesh 2.4&nbsp;— A 2.4&nbsp;GHz XBee module that uses DigiMesh, a sleeping mesh networking protocol developed by Digi International
* XBee-PRO DigiMesh 2.4&nbsp;— A higher power, longer range version of the XBee DigiMesh 2.4
* XBee ZB&nbsp;— An XBee module that incorporates the [[ZigBee]] PRO mesh networking protocol
* XBee-PRO ZB&nbsp;— A higher power, longer range version of the XBee ZB
* XBee ZB SMT&nbsp;— A surface mount XBee running the ZigBee protocol
* XBee-PRO ZB SMT&nbsp;— A higher power, longer range version of the XBee ZB SMT
* XBee SE&nbsp;— An XBee ZB module that incorporates the security cluster for the [[ZigBee Smart Energy]] public profile
* XBee-PRO SE&nbsp;— A higher power, longer range version of the XBee SE
* XBee-PRO 900HP&nbsp;— A 900&nbsp;MHz XBee-PRO module with up to 28 mile range with high-gain antenna that supports DigiMesh networking protocol
* XBee-PRO 900 (Legacy)&nbsp;— A [[33-centimeter band|900&nbsp;MHz]] proprietary point-to-point and star topology module, not recommended for new design
* XBee-PRO XSC (S3B)&nbsp;—  A 900&nbsp;MHz module compatible over the air with the Digi 9XStream radios
* XBee-PRO DigiMesh 900 (Legacy)&nbsp;— A 900&nbsp;MHz module that uses DigiMesh, not recommended for new design (see XBee-PRO 900HP for new designs)
* XBee-PRO 868&nbsp;— An [[ZigBee#Radio hardware|868&nbsp;MHz]] 500&nbsp;mW long-range module that supports proprietary point-to-point and star, for use in Europe
* XBee 865/868LP&nbsp;— An 868&nbsp;MHz XBee module that uses DigiMesh, available in Surface Mount form-factor (also configurable to [[865&nbsp;MHz]] for use in India)
* XBee ZigBee (S2C)&nbsp;— Incorporates an upgrade to the transceiver chip, replacing the [[Silicon Labs]] EM250 with the Silicon Labs EM357, effectively adding more RAM, more flash, faster clock speed and lowering the current draw.<ref>http://www.digikey.ca/en/pdf/d/digi-international/digi-xbee-zigbee-new-product-notice</ref>
* XBee-PRO ZigBee (S2C)&nbsp;— A higher power, longer range version of the XBee ZigBee (S2C)
