'''Kubeflow''' is a [[Free software|free]] and [[open-source software|open-source]] platform for [[machine learning]] on [[Kubernetes]]. The Kubeflow project has multiple distinct [[Component-based software engineering|software components]] which each address specific stages of the [[MLOps|machine learning lifecycle]], including model development ''([[#Kubeflow Notebooks|Kubeflow Notebooks]]<ref name="kubeflow_notebooks" />)'', model training ''([[#Kubeflow Pipelines|Kubeflow Pipelines]],<ref name="kubeflow_pipelines" /> [[#Kubeflow Training Operator|Kubeflow Training Operator]]<ref name="kubeflow_training_operator" />)'', model serving ''([[#KServe|KServe]]{{efn|name=kserve_rename}}<ref name="kubeflow_kserve" />)'', and [[automated machine learning]] ''([[#Katib|Katib]]<ref name="kubeflow_katib" />)''.

Each component of Kubeflow can be deployed separately, and it is not a requirement to deploy every component.<ref name="kubeflow_install" />

== History ==

The Kubeflow project was first announced at ''[[Cloud Native Computing Foundation|KubeCon + CloudNativeCon North America 2017]]'' by [[Google]] engineers David Aronchick, Jeremy Lewi, and Vishnu Kannan<ref>{{Cite web |title="Hot Dogs or Not" - At Scale with Kubernetes [I] - Vish Kannan & David Aronchick, Google |url=https://www.youtube.com/watch?v=R3dVF5wWz-g |language=en}}</ref> to address a perceived lack of flexible options for building production-ready machine learning systems.<ref>{{Cite web |title=Introducing Kubeflow - A Composable, Portable, Scalable ML Stack Built for Kubernetes |url=https://kubernetes.io/blog/2017/12/introducing-kubeflow-composable/ |language=en}}</ref> The project has also stated it began as a way for Google to open-source how they ran [[TensorFlow]] internally.<ref name="kubeflow_history" />

The first release of Kubeflow (Kubeflow 0.1) was announced at ''KubeCon + CloudNativeCon Europe 2018''<ref>{{Cite web |title=Google-led Kubeflow, machine learning for Kubernetes, begins to take shape |url=https://techcrunch.com/2018/05/04/google-kubeflow-machine-learning-for-kubernetes-begins-to-take-shape/ |language=en}}</ref> with claims of having already become among the top 2% of [[GitHub]] projects ever.<ref>{{Cite web |title=Announcing Kubeflow 0.1 |url=https://kubernetes.io/blog/2018/05/04/announcing-kubeflow-0.1/ |language=en}}</ref> Kubeflow 1.0 was released in March 2020 via a public blog post announcing that many Kubeflow components were graduating to a "stable status", indicating they were now ready for production usage.<ref>{{Cite web |title=Kubeflow 1.0: Cloud-Native ML for Everyone |url=https://blog.kubeflow.org/releases/2020/03/02/kubeflow-1-0-cloud-native-ml-for-everyone.html |language=en}}</ref>

== Components ==

=== Kubeflow Notebooks ===

The ''Kubeflow Notebooks'' component provides a way to run web-based development environments inside a Kubernetes cluster, with native support for [[Jupyter Notebook]], [[Visual Studio Code]], and [[RStudio]].<ref name="kubeflow_notebooks_overview" />

=== Kubeflow Pipelines ===

The ''Kubeflow Pipelines'' component provides a platform for building and deploying [[Software portability|portable]], scalable machine learning workflows based on Docker containers.<ref name="kubeflow_pipelines_intro" /> [[Google Cloud Platform]] has adopted the ''Kubeflow Pipelines DSL'' within its ''Vertex AI Pipelines'' product.<ref>{{Cite web |title=Vertex AI - Building a pipeline |url=https://cloud.google.com/vertex-ai/docs/pipelines/build-pipeline |language=en}}</ref>

=== Kubeflow Training Operator ===

The ''Kubeflow Training Operator'' component provides [[Kubernetes#Operators|Kubernetes custom resources]] that make it easy to run distributed or non-distributed [[TensorFlow]], [[PyTorch]], [[Apache MXNet]], [[XGBoost]], and [[Open MPI|MPI]] training jobs on Kubernetes.<ref name="kubeflow_training_operator" />

=== KServe ===

The ''KServe'' component (previously named KFServing<ref name="kubeflow_kserve_rename" />) provides [[Kubernetes#Operators|Kubernetes custom resources]] for serving machine learning models on arbitrary frameworks including [[TensorFlow]], [[XGBoost]], [[scikit-learn]], [[PyTorch]], and [[Open Neural Network Exchange|ONNX]].<ref>{{Cite web |title=KServe GitHub |url=https://github.com/kserve/kserve |language=en}}</ref> KServe was developed collaboratively by [[Google]], [[IBM]], [[Bloomberg L.P.|Bloomberg]], [[NVIDIA]], and [[Seldon (company)|Seldon]].<ref name="kubeflow_kserve_rename" /> Publicly disclosed adopters of KServe include [[Bloomberg L.P.|Bloomberg]],<ref>{{Cite web |title=The journey to build Bloombergâ€™s ML Inference Platform Using KServe (formerly KFServing) |url=https://www.bloomberg.com/company/stories/the-journey-to-build-bloombergs-ml-inference-platform-using-kserve-formerly-kfserving/ |language=en}}</ref> [[Gojek]],<ref>{{Cite web |title=Merlin: Making ML Model Deployments Magical |url=https://www.gojek.io/blog/merlin-making-ml-model-deployments-magical |language=en}}</ref> and others<ref>{{Cite web |title=KServe Website - Adopters of KServe |url=https://kserve.github.io/website/master/community/adopters/ |language=en}}</ref>

=== Katib ===

The ''Katib'' component is described as a Kubernetes-native project for [[automated machine learning]] with support for [[Hyperparameter optimization|hyperparameter tuning]], [[early stopping]], and [[neural architecture search]].<ref>{{Cite web |title=Kubeflow GitHub - Katib |url=https://github.com/kubeflow/katib |language=en}}</ref>
