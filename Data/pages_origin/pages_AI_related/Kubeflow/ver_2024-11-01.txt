'''Kubeflow''' is an [[open-source software|open-source]] platform for [[machine learning]] and [[MLOps]] on [[Kubernetes]] introduced by [[Google]]. The different stages in a typical [[MLOps|machine learning lifecycle]] are represented with different [[Component-based software engineering|software components]] in Kubeflow, including model development ''([[#Kubeflow Notebooks|Kubeflow Notebooks]]<ref name="kubeflow_notebooks" />)'', model training ''([[#Kubeflow Pipelines|Kubeflow Pipelines]]'',''<ref name="kubeflow_pipelines" /> [[#Kubeflow Training Operator|Kubeflow Training Operator]]<ref name="kubeflow_training_operator" />)'', model serving ''([[#KServe|KServe]]{{efn|name=kserve_rename}}<ref name="kubeflow_kserve" />)'', and [[automated machine learning]] ''([[#Katib|Katib]]<ref name="kubeflow_katib" />)''.

Each component of Kubeflow can be deployed separately, and it is not a requirement to deploy every component.<ref name="kubeflow_install" />

== History ==

The Kubeflow project was first announced at ''[[Cloud Native Computing Foundation|KubeCon + CloudNativeCon North America 2017]]'' by [[Google]] engineers David Aronchick, Jeremy Lewi, and Vishnu Kannan<ref>{{Cite web |title="Hot Dogs or Not" - At Scale with Kubernetes [I] - Vish Kannan & David Aronchick, Google | website=[[YouTube]] |url=https://www.youtube.com/watch?v=R3dVF5wWz-g |language=en}}</ref> to address a perceived lack of flexible options for building production-ready machine learning systems.<ref>{{Cite web |title=Introducing Kubeflow - A Composable, Portable, Scalable ML Stack Built for Kubernetes |date=21 December 2017 |url=https://kubernetes.io/blog/2017/12/introducing-kubeflow-composable/ |language=en}}</ref> The project has also stated it began as a way for Google to open-source how they ran [[TensorFlow]] internally.<ref name="kubeflow_history" />

The first release of Kubeflow (Kubeflow 0.1) was announced at ''KubeCon + CloudNativeCon Europe 2018''<ref>{{Cite web |title=Google-led Kubeflow, machine learning for Kubernetes, begins to take shape |date=4 May 2018 |url=https://techcrunch.com/2018/05/04/google-kubeflow-machine-learning-for-kubernetes-begins-to-take-shape/ |language=en}}</ref>.<ref>{{Cite web |title=Announcing Kubeflow 0.1 |date=4 May 2018 |url=https://kubernetes.io/blog/2018/05/04/announcing-kubeflow-0.1/ |language=en}}</ref> Kubeflow 1.0 was released in March 2020 via a public blog post announcing that many Kubeflow components were graduating to a "stable status", indicating they were now ready for production usage.<ref>{{Cite web |title=Kubeflow 1.0: Cloud-Native ML for Everyone |date=2 March 2020 |url=https://blog.kubeflow.org/releases/2020/03/02/kubeflow-1-0-cloud-native-ml-for-everyone.html |language=en}}</ref>

In October 2022, Google announced that the Kubeflow project had applied to join the [[Cloud Native Computing Foundation]].<ref>{{Cite web |last=Lamkin |first=Thea |date=2022-10-24 |title=Kubeflow has applied to become a CNCF incubating project |url=https://blog.kubeflow.org/kubeflow-applied-cncf-incubating/ |access-date=2023-11-02 |website=Kubeflow |language=en}}</ref><ref>{{Cite web |date=2022-10-24 |title=Kubeflow applies to become a CNCF incubating project |url=https://opensource.googleblog.com/2022/10/kubeflow-applies-to-become-a-cncf-incubating-project.html |access-date=2023-11-02 |website=Google Open Source Blog}}</ref> In July 2023, the foundation voted to accept Kubeflow as an incubating stage project.<ref>{{Cite web |date=2023-07-25 |title=Kubeflow brings MLOps to the CNCF Incubator |url=https://www.cncf.io/blog/2023/07/25/kubeflow-brings-mlops-to-the-cncf-incubator/ |access-date=2023-11-02 |website=Cloud Native Computing Foundation |language=en-US}}</ref><ref>{{Cite web |date=2023-07-25 |title=Kubeflow joins the CNCF family |url=https://opensource.googleblog.com/2023/07/kubeflow-joins-cncf-family.html |access-date=2023-11-02 |website=Google Open Source Blog}}</ref>

== Components ==

=== ''Kubeflow Notebooks'' for model development{{anchor|Kubeflow Notebooks}} ===

Machine learning models are developed in the notebooks component called ''Kubeflow Notebooks''. The component runs web-based development environments inside a Kubernetes cluster, with native support for [[Jupyter Notebook]], [[Visual Studio Code]], and [[RStudio]].<ref name="kubeflow_notebooks_overview" />

=== ''Kubeflow Pipelines'' for model training{{anchor|Kubeflow Pipelines}} ===

Once developed, models are trained in the ''Kubeflow Pipelines'' component. The component acts as a platform for building and deploying [[Software portability|portable]], scalable machine learning workflows based on Docker containers.<ref name="kubeflow_pipelines_intro" /> [[Google Cloud Platform]] has adopted the ''Kubeflow Pipelines DSL'' within its ''Vertex AI Pipelines'' product.<ref>{{Cite web |title=Vertex AI - Building a pipeline |url=https://cloud.google.com/vertex-ai/docs/pipelines/build-pipeline |language=en}}</ref>

=== ''Kubeflow Training Operator'' for model training{{anchor|Kubeflow Training Operator}} ===

For certain machine learning models and libraries, the ''Kubeflow Training Operator'' component provides [[Kubernetes#Operators|Kubernetes custom resources]] support. The component runs distributed or non-distributed [[TensorFlow]], [[PyTorch]], [[Apache MXNet]], [[XGBoost]], and [[Open MPI|MPI]] training jobs on Kubernetes.<ref name="kubeflow_training_operator" />

=== ''KServe'' for model serving{{anchor|KServe}} ===

The ''KServe'' component (previously named KFServing<ref name="kubeflow_kserve_rename" />) provides [[Kubernetes#Operators|Kubernetes custom resources]] for serving machine learning models on arbitrary frameworks including [[TensorFlow]], [[XGBoost]], [[scikit-learn]], [[PyTorch]], and [[Open Neural Network Exchange|ONNX]].<ref>{{Cite web |title=KServe GitHub |website=[[GitHub]] |url=https://github.com/kserve/kserve |language=en}}</ref> KServe was developed collaboratively by [[Google]], [[IBM]], [[Bloomberg L.P.|Bloomberg]], [[NVIDIA]], and [[Seldon (company)|Seldon]].<ref name="kubeflow_kserve_rename" /> Publicly disclosed adopters of KServe include [[Bloomberg L.P.|Bloomberg]],<ref>{{Cite news |title=The journey to build Bloomberg's ML Inference Platform Using KServe (formerly KFServing) |newspaper=Bloomberg L.p. |date=12 October 2021 |url=https://www.bloomberg.com/company/stories/the-journey-to-build-bloombergs-ml-inference-platform-using-kserve-formerly-kfserving/ |language=en}}</ref> [[Gojek]],<ref>{{Cite web |title=Merlin: Making ML Model Deployments Magical |url=https://www.gojek.io/blog/merlin-making-ml-model-deployments-magical |language=en}}</ref> the [[Wikimedia Foundation]],<ref>{{Cite web |title=Machine Learning/LiftWing |url=https://wikitech.wikimedia.org/wiki/Machine_Learning/LiftWing |language=en}}</ref> and others.<ref>{{Cite web |title=KServe Website - Adopters of KServe |url=https://kserve.github.io/website/master/community/adopters/ |language=en}}</ref>

=== ''Katib'' for automated machine learning{{anchor|Katib}} ===

Lastly, Kubeflow includes a component for [[Automated machine learning|automated training and development of machine learning models]], the ''Katib'' component. It is described as a Kubernetes-native project and features [[Hyperparameter optimization|hyperparameter tuning]], [[early stopping]], and [[neural architecture search]].<ref>{{Cite web |title=Kubeflow GitHub - Katib |website=[[GitHub]] |url=https://github.com/kubeflow/katib |language=en}}</ref>
